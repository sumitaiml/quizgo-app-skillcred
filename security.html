Security & Core Privacy

SPLINTR operates on strict stateless security protocols. When a curriculum document is uploaded, it is parsed and processed in memory. All temporary files are instantly destroyed post-synthesis, ensuring zero-latency data footprint preservation.

User credentials and quiz archives are stored in encrypted Supabase PostgreSQL vaults, secured by service-role authentication policies. Your data is yours, isolated from external training sets, and never used for machine learning models.

By enforcing row-level security (RLS) policies within the database layer, only the authenticated generator can access historical battle statistics and quiz logs.

ENCRYPTION LEVEL AES-256

// SECURITY ARCHITECTURE PROTOCOLS

01. Stateless Processing

We do not store your raw curriculum text. The PDF is converted to content streams in memory, analyzed via the Gemini API securely, and immediately discarded.

02. Row Level Isolation

Historical quizzes are connected to your unique Supabase Auth UUID. No user can view, edit, or access another developer's telemetry logs.
